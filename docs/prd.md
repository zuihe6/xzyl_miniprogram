## 四、各功能模块详细设计

### 4.1 旅游租车

#### 4.1.1 车型列表页

- **顶部**：车型分类 Tab，包含但不限于：
  - 5座越野
  - 5座轿车
  - 7-9座商务
- **用车城市**：下拉框，可选城市：
  > 拉萨、林芝、阿里、日喀则、山南、那曲、波密、昌都
- **车辆列表**（每行一条，左图右文）：

  | 位置 | 内容 |
  |------|------|
  | 左侧 | 车辆图片 |
  | 右侧第一行 | 车辆描述 + 收藏按钮 |
  | 右侧第二行 | 简短特点 |
  | 右侧第三行 | 车辆品牌（加黄色背景显示） |
  | 右侧第四行 | 价格（￥xxx 元起） |

#### 4.1.2 车辆详情页

页面内容从上到下：

1. 车辆图片
2. 车辆名称
3. 价格 + 品牌
4. 推荐理由 + 简短特点（加黄色背景）
5. 产品说明 + 预定须知

**产品说明 — 服务流程步骤：**

```
提交订单 → 确认订单（客服主动联系沟通形成）
→ 支付订单确认接单 → 安排司导优先匹配
→ 上门接驾准时就位 → 开启行程可灵活调整
```

**费用包含：**
- 车辆服务费；燃油/电费
- 司导服务费
- 司导小费
- 停车费、高速费、路桥费

**费用不含：**
- 司导餐食和住宿补助
- 超时长费：单日上限 10 小时，超时费用下单时请与销售确认
- 超公里费：单日上限 200 公里，超公里费用下单时请与销售确认
- 其他用车过程中产生的额外费用，以实际产生为准，线下支付给司机的部分不接受开票

**底部固定栏：**
> 合计：￥xxx 起 ｜ 【立即购买】按钮

#### 4.1.3 立即购买 — 购买页面

**第一行（车辆信息）：**
- 左：车辆图片
- 右第一行：车辆名称
- 右第二行：购买数量

**第二行（填写信息，除"其他需求"外均为必填）：**

| 字段 | 是否必填 |
|------|----------|
| 姓名 | ✅ 必填 |
| 手机号 | ✅ 必填 |
| 用车城市 | ✅ 必填 |
| 接驳地址 | ✅ 必填 |
| 用车时间 | ✅ 必填 |
| 用车天数 | ✅ 必填 |
| 用车人数 | ✅ 必填 |
| 其他需求 | 选填 |

**底部固定栏：**
> 合计：￥xx 元 ｜ 明细 ｜ 【提交订单】按钮
>
> 点击"明细"弹出费用明细说明

### 4.3 定制游

#### 第一部分：基本信息

> 标题样式：**`01`** 添加强调色 ｜ **基本信息**（加粗）

| 行 | 内容 |
|----|------|
| 第一行 | 标题：01 基本信息 |
| 第二行 | 标签：出发地 / 目的地 |
| 第三行 | 输入框：出发地 / 目的地 |
| 第四行 | 标签：出发日期 |
| 第五行 | 日期选择器（开始日期 / 结束日期），规则：结束日期不早于开始日期，不晚于当前时间，支持切换上/下月 |
| 第六行 | 标签：游玩天数 / 人均预算（元） |
| 第七行 | 输入框：天数 / 预算 |
| 第八行 | 标签：成人 / 儿童 |
| 第九行 | 计数器（可加减）：成人数量 / 儿童数量 |

#### 第二部分：特色定制

| 行 | 内容 |
|----|------|
| 第一行 | 标签：酒店星级 |
| 第二行 | 选项：三星/舒适 ｜ 四星/高档 ｜ 五星/豪华 |
| 第三行 | 选项：连锁酒店 ｜ 特色客栈 ｜ 不限 |
| 第四行 | 标签：用餐要求 |
|  | 选项：当地特色 ｜ 定制师推荐 ｜ 自理 |
| 第五行 | 标签：其他需求 |
| 第六行 | 文本输入框 |

#### 第三部分：联系人信息

- 姓名（必填）
- 手机号码（必填）

**底部：**
> 【提交定制】按钮

**顶部信息区：**
- 左：景区图片

- 手机号码（必填）

**产品介绍区（与上方区块留间距）：**
- 显示产品详情介绍

**底部固定栏：**
> 合计：￥xxx 起 ｜ 明细 ｜ 【立即购买】按钮

## 八、非功能性需求

### 8.1 UI/UX 规范

| 项目 | 要求 |
|------|------|
| 组件库 | 强制使用 **TDesign WeiXin**（[官方文档](https://tdesign.tencent.com/miniprogram/getting-started)） |
| 风格 | 统一、简洁、美观 |
| 交互 | 流畅，列表滑动、页面切换不卡顿 |

### 8.2 性能要求

- 小程序首屏打开速度要快，建议优化图片懒加载、分包加载等

### 8.3 安全要求

- 微信登录流程需安全，服务端校验 `code`，不在前端暴露敏感信息
- 手机号授权通过微信官方 `getPhoneNumber` API 获取，不允许明文传输
- 用户信息在展示时中间号码打星号处理（如：138****8888）
